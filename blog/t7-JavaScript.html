Ruby Hashes vs. JavaScript Objects

November 23, 2014

Objects can contain many values in JavaScript. For ex:
var food = {appetizer:"salad", dinner:"steak", dessert:"pie"};
The values are written in name:value pairs. The name and value is separated by a colon in Javascript. These pairs are called properties.

Ruby hashes can be constructed in a couple of different ways:


food = { "appetizer" => "salad", "dinner" => "steak", "dessert" => "pie" }
If the keys are always symbols, a hash can be written like this:
food = { :appetizer => "salad", :dinner => "steak", :dessert => "pie" }
You can also write hashes like this as well:
food = { appetizer: "salad", dinner: "steak", dessert: "pie" }

The difference between the two is that hashes in ruby can be written in a couple of different ways, and keys can be symbols. In Ruby we call the pairing of a hash key / value pairs, and in JavaScript it's a name / value pair.
